## Key Components

### 1. LMDB Conversion (`convert_to_lmdb.py`)

Before using the optimized dataset, the raw graph data (stored in `batch_*.pt` files, likely generated by `src/simulation/phenotype_simulation/create_hpo_dataset.py`) needs to be converted into an LMDB (Lightning Memory-Mapped Database). This is a **one-time process** that significantly speeds up data loading, especially random access required during training.

**Usage:**

```bash
# Ensure the 'processed' directory with batch_*.pt and index.pt exists
# Example: python -m training.datasets.convert_to_lmdb /path/to/dataset/root --batch-commit 500
python -m training.datasets.convert_to_lmdb [DATASET_ROOT_DIR] --batch-commit [COMMIT_SIZE]
```

*   `[DATASET_ROOT_DIR]`: Path to the dataset root directory (e.g., `data/simulation/output/`). The script expects `[DATASET_ROOT_DIR]/processed/index.pt` and `[DATASET_ROOT_DIR]/processed/batch_*.pt`.
*   `[COMMIT_SIZE]`: Number of samples to write to LMDB per transaction (e.g., 500 or 5000). Larger values can be faster but use more RAM during conversion.

This script creates the `[DATASET_ROOT_DIR]/hpo_lmdb/` directory containing `data.mdb` and `lock.mdb`.

### 2. LMDB Database Structure (`hpo_lmdb/data.mdb`)

The `data.mdb` file is a key-value store with the following structure:

*   **Data Keys:** Integer indices (from 0 to N-1, where N is the total number of samples) converted to ASCII byte strings (e.g., `b'0'`, `b'1'`, `b'5153271'`).
*   **Data Values:** The corresponding `torch_geometric.data.Data` object for that sample, serialized using `torch.save` into bytes.
*   **Metadata Key `b'__len__'`:** Stores the total number of samples (N) as an ASCII-encoded string.
*   **Metadata Key `b'__gene_idx_map__'`:** Stores a Python dictionary (serialized using `pickle`) that maps gene name strings to lists of integer indices belonging to that gene. This allows for efficient retrieval of samples by gene.

### 3. LMDB Dataset Class (`LMDBHPOGraphDataset`)

The `LMDBHPOGraphDataset` class (in `lmdb_hpo_dataset.py`) provides a standard PyTorch Geometric `Dataset` interface for efficiently reading data from the pre-built LMDB database.

```python
from training.datasets.lmdb_hpo_dataset import LMDBHPOGraphDataset
from torch_geometric.loader import DataLoader as PyGDataLoader

# Initialize dataset (reads from hpo_lmdb within the root_dir)
dataset_root = "data/simulation/output/" 
try:
    dataset = LMDBHPOGraphDataset(root_dir=dataset_root, readonly=True)
    print(f"Loaded dataset with {len(dataset)} samples.")

    # Access individual samples by index
    sample_index = 12345
    sample = dataset[sample_index] # Uses get(idx) internally
    print(f"Sample {sample_index} - Gene: {sample.gene}")
    print(f"Number of nodes: {sample.x.shape[0]}")

    # Get mask for a specific gene (uses the __gene_idx_map__ in LMDB)
    gene_mask = dataset.get_gene_mask('SOME_GENE')
    gene_indices = torch.where(gene_mask)[0]
    print(f"Found {len(gene_indices)} samples for SOME_GENE.")

except FileNotFoundError:
    print(f"LMDB database not found at {dataset_root}/hpo_lmdb. Run convert_to_lmdb.py first.")
except Exception as e:
    print(f"Error loading dataset: {e}")

```

### Splitting Strategies

Splitting the `LMDBHPOGraphDataset` primarily involves working with the dataset indices ranging from `0` to `len(dataset) - 1`.

1.  **Random Split**: Use `torch.utils.data.random_split`.

    ```python
    import torch
    from torch.utils.data import random_split, Subset

    # Assuming 'dataset' is an initialized LMDBHPOGraphDataset
    n_total = len(dataset)
    n_train = int(0.7 * n_total)
    n_val = int(0.15 * n_total)
    n_test = n_total - n_train - n_val

    torch.manual_seed(42)
    train_dataset, val_dataset, test_dataset = random_split(dataset, [n_train, n_val, n_test])

    # train_dataset, val_dataset, test_dataset are Subset objects
    # containing the original dataset and the relevant indices.
    print(f"Train samples: {len(train_dataset)}") 
    ```

2.  **Gene Stratified Split**: Requires custom logic using `get_gene_mask`. You would typically iterate through genes, get indices for each using `get_gene_mask`, split those indices, and then combine the results. The function `training.dataloaders.splitting.gene_stratified_split` may need adaptation for the LMDB dataset.

3.  **Precomputed Split**: If split index files (`train_indices.pt`, etc.) exist (e.g., in `[DATASET_ROOT_DIR]/splits/`), you can load them.

    ```python
    import torch
    from torch.utils.data import Subset

    # Assuming 'dataset' is an initialized LMDBHPOGraphDataset
    splits_dir = dataset.root_dir / "splits" 
    train_indices = torch.load(splits_dir / "train_indices.pt")
    val_indices = torch.load(splits_dir / "val_indices.pt")

    train_subset = Subset(dataset, train_indices)
    val_subset = Subset(dataset, val_indices)
    ```

```python

from torch.utils.data import Subset
import torch # For loading indices

# Assuming 'dataset' is an initialized LMDBHPOGraphDataset

train_loader_full = PyGDataLoader(
    dataset,
    batch_size=32,
    shuffle=True, 
    num_workers=4,
    pin_memory=True
)

# Option 2: Load precomputed split using Subset

train_indices = torch.load(splits_dir / "train_indices.pt")
```

2.  Runs a basic speed test using `PyGDataLoader` with shuffling (if dataset loads).
3.  Performs direct LMDB inspection to check specific keys and data structure.
